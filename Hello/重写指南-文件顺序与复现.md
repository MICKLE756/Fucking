# Hello-Agents 复现指南 · 文件编写顺序（自底向上）

> 目的：给出**从零复现整个 `Hello/hello_agents` 项目**时「先写哪个文件、后写哪个文件」的精确顺序。
> 本文是**顺序清单/施工图**，偏"按什么次序敲文件、每写完一层怎么自测"；每个文件的**具体实现细节**请对照同目录下的
> [`第七章框架重写指南.md`](第七章框架重写指南.md)（1300+ 行详解）与参考实现 `../Hello-Agents/hello_agents/` 逐行照抄。
>
> 核心原则：**依赖在下、被依赖在上，自底向上写**。每写完一个阶段就跑该阶段的「✅ 自测」，绿了再进入下一阶段。

---

## 总览：8 个阶段

| 阶段 | 目录 | 内容 | 对应章节 | 依赖 |
|---|---|---|---|---|
| 0 | 项目骨架 | 目录 / `.env` / `requirements.txt` | — | 无 |
| 1 | `core/` | LLM、消息、配置、Agent 基类等地基 | 第七章 | 阶段 0 |
| 2 | `tools/` | 工具基类、注册表、搜索/计算器、工具链 | 第七章 | 阶段 1 |
| 3 | `agents/` | 5 种 Agent 范式 | 第七章 | 阶段 1、2 |
| 4 | `memory/` | 嵌入、四种记忆、存储、RAG | 第八章 | 阶段 1、2 |
| 5 | `context/` | 上下文工程（构建/裁剪/计数/历史） | 第九章 | 阶段 1、2、4 |
| 6 | `protocols/` | MCP / A2A / ANP 通信协议 | 第十章 | 阶段 1、2 |
| 7 | `rl/` + `evaluation/` | 强化学习 + 性能评估 | 第十一/十二章 | 阶段 1、2、3 |
| 8 | `web/` | Agent 流程可视化前端（本仓库扩展） | 扩展 | 阶段 1、2、3 |

> **说明**：阶段 1~3 是**必做地基**；阶段 4 起为按需扩展模块（依赖可选重型库，缺失时相关 `__init__` 用 `try/except` 优雅跳过，不影响 `import hello_agents`）。

## 阶段 0 · 项目骨架

先建目录与占位 `__init__.py`（内容后面阶段再填），再写依赖与环境文件。

1. `version.py` — 版本号/作者等元信息（被顶层 `__init__.py` 引用，先写）。
2. `requirements.txt` — 核心依赖 `openai / pydantic / python-dotenv / requests`；可选重依赖（qdrant-client、sentence-transformers、fastmcp、trl、datasets…）分组注释。
3. `.env`（**不入库**）— 参考 `../Hello-Agents/.env.example`，至少：
   ```
   LLM_MODEL_ID=...        # 任意 OpenAI 兼容模型
   LLM_API_KEY=...
   LLM_BASE_URL=...
   # 第八章 RAG（可选）
   EMBED_MODEL_TYPE=local
   EMBED_MODEL_NAME=C:/Files/Models/bge-base-zh-v1.5
   QDRANT_URL=http://localhost:6333
   ```

> 顶层 `hello_agents/__init__.py` 与各子包 `__init__.py` 的**导出内容留到对应阶段末尾再补**——因为它们 import 的符号要等文件写完才存在。

## 阶段 1 · `core/`（第七章地基，严格按序）

> 顺序即依赖顺序：下面的文件只依赖它上面的。

1. `core/exceptions.py` — `HelloAgentsException` 等异常，最底层、无依赖。
2. `core/message.py` — `Message` / `MessageRole`（`Literal["user","assistant","system","tool","summary"]`，`summary` 供第九章历史压缩用）。
3. `core/config.py` — `Config`（temperature / max_tokens / debug 等）。
4. `core/llm_response.py` — `LLMResponse / StreamStats / ToolCall / LLMToolResponse` 数据契约。
5. `core/llm.py` — `HelloAgentsLLM`：和大模型对话的唯一出口（含超时/重试）。依赖 1-4。
6. `core/lifecycle.py` — `EventType / AgentEvent / ExecutionContext / LifecycleHook` 生命周期钩子。
7. `core/streaming.py` — `StreamEvent(Type) / StreamBuffer / stream_to_sse / stream_to_json` 流式输出。
8. `core/session_store.py` — `SessionStore` 会话持久化。
9. `core/agent.py` — `Agent` 抽象基类（所有 Agent 的共同父类）。依赖 message/config/llm。
10. `core/database_config.py` — 数据库连接配置（第八章存储用，可与阶段 4 一起写）。
11. **补** `core/__init__.py` — 导出上述符号（见 `../Hello-Agents/hello_agents/core/__init__.py`）。

✅ 自测：`python -c "from hello_agents.core import Agent, HelloAgentsLLM, Message, Config"`

---

## 阶段 2 · `tools/`（第七章）

1. `tools/base.py` — `Tool`（抽象基类）+ `ToolParameter`。
2. `tools/registry.py` — `ToolRegistry / global_registry`。依赖 base。
3. `tools/builtin/search_tool.py` — `SearchTool`（轻量，仅核心库）。
4. `tools/builtin/calculator.py` — `CalculatorTool`（轻量）。
5. `tools/chain.py` — `ToolChain / ToolChainManager / create_research_chain / create_simple_chain`。
6. `tools/async_executor.py` — `AsyncToolExecutor / run_parallel_tools(_sync) / run_batch_tool(_sync)`。
7. **补** `tools/__init__.py` 与 `tools/builtin/__init__.py` — 先只导出 base/registry/search/calculator/chain/async_executor；**其余重型工具（memory/rag/note/terminal/protocol/评估/rl）在各自阶段完成后，用 `try/except` 追加导出**（缺依赖时优雅跳过）。

✅ 自测：`python -c "from hello_agents.tools import SearchTool, CalculatorTool, ToolRegistry"`

---

## 阶段 3 · `agents/`（第七章，5 种范式）

顺序（互不强依赖，建议由简到繁）：
1. `agents/simple_agent.py` — `SimpleAgent`
2. `agents/react_agent.py` — `ReActAgent`（思考→行动→观察循环，前端可视化就基于它）
3. `agents/reflection_agent.py` — `ReflectionAgent`
4. `agents/plan_solve_agent.py` — `PlanAndSolveAgent`
5. `agents/function_call_agent.py` — `FunctionCallAgent`
6. `agents/tool_aware_agent.py` — `ToolAwareSimpleAgent`
7. **补** `agents/__init__.py` 导出以上 6 个。
8. **补** 顶层 `hello_agents/__init__.py` — 导出 core + agents + 轻量 tools（此时地基完成，`import hello_agents` 可用）。

✅ 自测：`python -c "import hello_agents as h; print(h.__version__); h.SimpleAgent, h.ReActAgent"`

> 至此**第七章核心框架完成**，后续为按需扩展模块。

## 阶段 4 · `memory/`（第八章：记忆 + RAG）

1. `memory/base.py` — 记忆基类/接口。
2. `memory/embedding.py` — 嵌入统一层：`LocalTransformerEmbedding`(bge, L2 归一化, `is_query` 前缀) / `DashScopeEmbedding` / `TFIDFEmbedding` + 工厂 `get_text_embedder/get_dimension/refresh_embedder`。**所有 import 置于文件顶部，类 `__init__` 不写死模型名，模型名一律由 `.env` 经工厂传入。**
3. `memory/storage/document_store.py` — 文档存储。
4. `memory/storage/qdrant_store.py` — Qdrant 向量库（余弦召回）。
5. `memory/storage/neo4j_store.py` — 图存储（语义记忆用）。
6. `memory/storage/__init__.py` — 导出存储类。
7. `memory/types/working.py` → `episodic.py` → `semantic.py` → `perceptual.py` — **四种记忆类型**。
8. `memory/types/__init__.py` — 导出 `WorkingMemory / EpisodicMemory / SemanticMemory / PerceptualMemory` + 辅助类。
9. `memory/rag/document.py` — 文档切块/解析。
10. `memory/rag/pipeline.py` — RAG 管线：入库 `index_chunks`、查询 `embed_query(query, is_query=True)` + top_k 召回。
11. `memory/rag/__init__.py`、`memory/manager.py`、`memory/__init__.py` — 导出与统一管理器。
12. `tools/builtin/memory_tool.py`、`tools/builtin/rag_tool.py` — 把记忆/RAG 封装成工具，并在 `tools/__init__.py` 追加 `try/except` 导出。

✅ 自测（需本地 Qdrant + 嵌入模型）：`from hello_agents.memory.types import WorkingMemory, EpisodicMemory, SemanticMemory, PerceptualMemory`；跑 `examples/chapter08_memory_rag.py`。

## 阶段 5 · `context/`（第九章：上下文工程）

1. `context/token_counter.py` — token 计数（tiktoken）。
2. `context/truncator.py` — 上下文裁剪策略。
3. `context/history.py` — 历史管理（会产出 `role="summary"` 的压缩消息）。
4. `context/builder.py` — 上下文构建器（可组合 MemoryTool/RAGTool，用子模块绝对路径导入避免循环）。
5. `context/__init__.py` — 导出。

✅ 自测：`python -c "from hello_agents.context import *"`；跑 `examples/chapter09_context_engineering.py`。

## 阶段 6 · `protocols/`（第十章：通信协议）

1. `protocols/base.py` — 协议基类。
2. `protocols/mcp/{utils,client,server}.py` + `__init__.py` — MCP（需 `mcp`/`fastmcp`）。
3. `protocols/a2a/implementation.py` + `__init__.py` — A2A。
4. `protocols/anp/implementation.py` + `__init__.py` — ANP。
5. `protocols/__init__.py` — 导出（可选依赖缺失时优雅降级）。
6. `tools/builtin/protocol_tools.py`（`MCPTool/A2ATool/ANPTool`）、`tools/builtin/mcp_wrapper_tool.py`，并在 `tools/__init__.py` 追加导出。

✅ 自测：跑 `examples/chapter10_protocols.py`。

## 阶段 7 · `rl/` + `evaluation/`（第十一/十二章）

**RL（第十一章）**
1. `rl/utils.py` → `rl/rewards.py` → `rl/datasets.py` → `rl/trainers.py`（`datasets`/`trl` 可选）。
2. `rl/__init__.py`；`tools/builtin/rl_training_tool.py` + `tools/__init__.py` 追加导出。

**评估（第十二章）**
3. `evaluation/benchmarks/bfcl/{dataset,evaluator,metrics,bfcl_integration}.py`
4. `evaluation/benchmarks/gaia/{dataset,evaluator,metrics}.py`
5. `evaluation/benchmarks/data_generation/{dataset,llm_judge,win_rate}.py`
6. 各级 `__init__.py`；工具 `tools/builtin/{bfcl_evaluation_tool,gaia_evaluation_tool,llm_judge_tool,win_rate_tool,note_tool,terminal_tool}.py` + `tools/__init__.py` 追加导出。

✅ 自测：`examples/chapter11_RL.py`；`python -c "import hello_agents.evaluation"`。

## 阶段 8 · `web/`（Agent 流程可视化前端 · 本仓库扩展）

1. `web/tracer.py` — `TraceEvent` 数据类 + `TracedReActAgent`（子类，逐步 `emit`）+ `demo_trace()`（无需 LLM 的演示流）。依赖 `agents/react_agent.py`。
2. `web/server.py` — FastAPI：`GET /`、`GET /api/health`、`GET /api/run?q=..&mode=demo|live`（SSE 逐条推送 `TraceEvent`）。
3. `web/static/index.html` — 单页前端：按事件类型着色的时间线（思考/行动/观察/最终答案）。
4. `web/__init__.py`。

✅ 自测：`python -m hello_agents.web.server` → 浏览器开 `http://127.0.0.1:8080` → 选「演示模式」运行 → 时间线完整渲染出「思考→行动→观察→最终答案」。
