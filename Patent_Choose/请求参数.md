# 请求参数

本文档整理成员4检索服务需要从上游模块获取的参数，来源于《成员4接口规范文档》。

## 主检索接口

- 接口地址：`POST /retrieve/search`
- 主要调用方：意图识别与交互编排模块、业务服务内部调用方
- 说明：该接口是主检索链路入口，上游需要至少提供非空 `query`。

### 请求头

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| X-Request-ID | string | 否 | 请求链路 ID；不传时服务会自动生成 UUID，并在响应头返回同名字段 |

### 请求体参数

| 参数名 | 类型 | 必填 | 约束/默认值 | 需要上游提供的内容 |
|--------|------|------|-------------|--------------------|
| query | string | 是 | 最小长度 1 | 用户原始检索文本 |
| session_id | string | 否 | 默认空 | 会话 ID，用于日志追踪 |
| tech_field | string | 否 | 默认空 | 技术领域，会参与 Query 规范化 |
| core_problem | string | 否 | 默认空 | 核心技术问题，会参与 Query 规范化 |
| constraints | object | 否 | 默认空 | 结构化约束条件，会拼接到规范化 Query |
| filters | object | 否 | 默认空 | 字段过滤条件 |
| sort_preference | string | 否 | 当前接收但未生效 | 排序偏好预留字段 |
| top_k | integer | 否 | 默认 10，范围 1-20 | 最终返回结果数量 |

### filters 子参数

| 参数名 | 类型 | 必填 | 需要上游提供的内容 | 当前实现说明 |
|--------|------|------|--------------------|--------------|
| tech_field | string | 否 | 需要过滤的技术领域 | 按召回结果 `tech_field` 做包含匹配 |
| maturity | string | 否 | 成熟度过滤条件 | 模型可接收，但当前未落库，过滤逻辑不生效 |
| publish_year_from | integer | 否 | 最早公开年份 | 按 `publish_date` 解析年份并过滤 |
| application_scene | string | 否 | 应用场景过滤条件 | 在 `title`、`tech_field`、`chunk_text` 拼接文本中做包含匹配 |

### constraints 常见子参数

| 参数名 | 类型 | 必填 | 需要上游提供的内容 |
|--------|------|------|--------------------|
| temperature | string | 否 | 温度约束，例如 `>=400℃` |
| application / scene | string | 否 | 应用场景 |
| material | string | 否 | 材料偏好 |
| maturity | string | 否 | 成熟度 |
| cost | string | 否 | 成本约束 |
| year | string/integer | 否 | 年份约束 |

未在映射表中的 `constraints` 键会按原键名拼接到规范化 Query。

### 请求示例

```json
{
  "query": "需要耐高温400度以上、用于餐饮厨具的不粘锅涂层材料",
  "session_id": "sess_001",
  "tech_field": "新材料",
  "core_problem": "耐高温环保不粘锅涂层",
  "constraints": {
    "temperature": ">=400℃",
    "scene": "餐饮厨具"
  },
  "filters": {
    "tech_field": "新材料",
    "publish_year_from": 2022,
    "application_scene": "厨具"
  },
  "sort_preference": "latest",
  "top_k": 10
}
```
