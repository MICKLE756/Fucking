# 请求参数

本文档整理专利智能检索系统的 HTTP 接口参数与内部状态参数规范。

---

## 一、对外 HTTP 接口

### 1.1 对话交互接口

- 接口地址：`POST /chat`
- 说明：用户对话入口，接收自然语言输入，经工作流引擎处理后返回智能回复

#### 请求体参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| message | string | 是 | 用户输入的自然语言文本 |

#### 请求示例

```json
{
  "message": "我想找一种耐高温的不粘锅涂层材料相关的专利"
}
```

#### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| message | string | 系统生成的自然语言回复 |
| request | string | 操作提示信息（可选，仅功能未上线时返回） |
| state | object | 当前会话状态快照（见 1.4 节） |

#### 响应示例

```json
{
  "message": "我理解您的需求是：寻找适用于餐饮厨具、满足耐温400℃以上的不粘涂层相关专利。请确认是否继续检索。",
  "state": {
    "phase": "confirming",
    "intent_level1": "search",
    "intent_level2": ["专利检索"],
    "tech_domain": "涂层、新材料",
    "core_problem": "耐高温不粘锅涂层",
    "constraints": {"temperature": "400", "application": "餐饮厨具"},
    "clarify_round": 0,
    "max_clarify_round": 3,
    "confirmed": false
  }
}
```

### 1.2 会话重置接口

- 接口地址：`POST /reset`
- 说明：清空当前会话状态，重新开始对话

#### 响应示例

```json
{
  "ok": true
}
```

### 1.3 首页

- 接口地址：`GET /`
- 说明：重定向到 `/static/index.html` 前端页面

### 1.4 响应中 state 字段说明

| 字段名 | 类型 | 说明 |
|--------|------|------|
| phase | string | 当前对话阶段（见第三节） |
| intent_level1 | string | 一级意图：search / analysis / operation / feedback / chitchat |
| intent_level2 | array[string] | 二级意图列表 |
| tech_domain | string | 已识别的技术领域 |
| core_problem | string | 已识别的核心技术问题 |
| constraints | object | 已收集的约束条件 |
| clarify_round | integer | 当前追问轮次 |
| max_clarify_round | integer | 最大追问轮次上限（默认 3） |
| confirmed | boolean | 用户是否已确认检索条件 |

## 二、内部工作流状态参数（state）

工作流引擎在各节点间传递的完整状态字典：

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| user_input | string | "" | 当前轮用户原始输入 |
| phase | string | "idle" | 对话阶段标记 |
| intent | dict | {} | 完整意图结构 `{一级意图: [二级意图列表]}` |
| intent_level1 | string | "" | 一级意图 |
| intent_level2 | array[string] | [] | 二级意图列表 |
| tech_domain | string | "" | 技术领域（实体抽取结果） |
| core_problem | string | "" | 核心问题（实体抽取结果） |
| constraints | object | {} | 约束条件集合 |
| clarify_round | integer | 0 | 已追问轮次 |
| max_clarify_round | integer | 3 | 最大追问轮次 |
| confirmed | boolean | false | 条件是否已确认 |
| history | array | [] | 对话历史 `[{role, content}, ...]` |
| _post_extract_route | string | "" | 实体抽取后路由标记：`"direct_execute"` 或 `"eval"` |
| _response | string | "" | 当前轮生成的回复文本 |
| _request | string | "" | 操作提示信息 |
| _eval_result | object | - | 完整性评估结果 |
| _tool_result | object | - | 工具执行结果 |
| _confirm_result | string | - | 确认检查结果：`"affirm"` / `"deny"` / `"not_confirming"` |

---

## 三、对话阶段（phase）枚举

| 值 | 说明 | 触发条件 |
|----|------|----------|
| idle | 空闲/初始 | 会话初始化或重置后 |
| recognized | 已识别意图 | 意图识别节点完成 |
| clarifying | 追问中 | 信息不完整，生成追问 |
| confirming | 等待确认 | 信息完整，等待用户确认 |
| executing | 执行中 | 用户确认后执行检索 |
| responding | 回复中 | 工具执行完毕，生成回复 |

## 四、意图体系

### 4.1 意图映射表（config.INTENT_INFO）

| 一级意图 | 二级意图 | 说明 |
|---------|---------|------|
| search | 专利检索 | 根据技术领域/核心问题检索专利 |
| search | 专利详情查询 | 根据专利号查看单条专利详情 |
| analysis | SWOT分析 | 对指定专利做 SWOT 分析 |
| analysis | 技术对比分析 | 对比两个或多个专利/方案 |
| analysis | 风险评估 | 侵权风险、法律风险分析 |
| analysis | 价值评估 | 商业价值、技术价值评估 |
| operation | 专利聚束组合 | 组合多个专利形成方案 |
| operation | 专利收藏 | 收藏指定专利（待实现） |
| operation | 导出报告 | 导出分析/检索报告（待实现） |
| feedback | 结果不满意 | 用户对当前结果不满意 |
| feedback | 修改条件 | 补充/修改检索约束 |
| feedback | 换方向 | 更换技术领域重新检索 |
| chitchat | 闲聊 | 问候、感谢等社交对话 |
| chitchat | 无关输入 | 与专利无关的输入 |

### 4.2 各意图所需实体（config.ENTITY_INFO）

| 二级意图 | 需要抽取的实体 |
|---------|---------------|
| 专利检索 | 技术领域、核心问题、约束条件 |
| 专利详情查询 | 专利号 |
| SWOT分析 | 专利号、技术领域 |
| 技术对比分析 | 专利号、技术领域 |
| 风险评估 | 专利号、技术领域 |
| 价值评估 | 专利号 |
| 专利聚束组合 | 技术领域、核心问题 |
| 专利收藏 | 专利号 |
| 导出报告 | 专利号 |
| 结果不满意 | （无需实体） |
| 修改条件 | 约束条件 |
| 换方向 | 技术领域 |
| 闲聊 | （无需实体） |
| 无关输入 | （无需实体） |

## 五、实体抽取参数

### 5.1 实体类型定义

| 实体类型 | 数据类型 | 说明 | 示例 |
|---------|---------|------|------|
| 技术领域 | string | 技术方向关键词 | "涂层"、"汽车减振器"、"电池" |
| 核心问题 | string | 具体技术问题描述 | "耐高温不粘涂层"、"减振器性能优化" |
| 约束条件 | object | 结构化约束键值对 | `{"temperature": "400", "application": "餐饮厨具"}` |
| 专利号 | string | CN 开头的专利编号 | "CN202520842474.4" |

### 5.2 约束条件常见子参数

| 键名 | 类型 | 说明 | 抽取方式 |
|------|------|------|----------|
| temperature | string | 温度约束值 | 正则匹配 `(\d+)\s*[°℃度]` |
| application | string | 应用场景 | 关键词匹配 |
| material | string | 材料类型 | 关键词匹配 |
| patent_type | string | 专利类型（发明/实用新型） | 关键词匹配 |
| legal_status | string | 法律状态（已授权/有效） | 关键词匹配 |
| time_range | string | 时间范围 | 正则匹配年份模式 |
| patent_id | string | 专利号（内部传递） | 正则匹配 `CN\d{9,13}[\.\-]?\d?` |

## 七、意图识别流水线参数

三级 Fallback 管道：规则 → 远程微调模型 → OpenAI 兜底 → 状态机修正

### 7.1 远程微调模型配置（.env）

| 环境变量 | 类型 | 默认值 | 说明 |
|---------|------|--------|------|
| REMOTE_MODEL_URL | string | "" | vLLM 部署地址，如 `http://localhost:8000/v1` |
| REMOTE_MODEL_NAME | string | "" | 模型名称，如 `Qwen2.5-8B-sft-all` |
| REMOTE_MODEL_API_KEY | string | "" | API Key（无鉴权时留空） |
| REMOTE_MODEL_TIMEOUT | float | 5 | 请求超时秒数 |

未配置 `REMOTE_MODEL_URL` 时自动跳过远程模型分支。

### 7.2 远程模型请求格式

```json
{
  "model": "Qwen2.5-8B-sft-all",
  "messages": [
    {"role": "system", "content": "你是一个专利领域意图分类器。根据用户输入判断其意图，以JSON格式输出。"},
    {"role": "user", "content": "我想找一种汽车减振器相关的专利"}
  ],
  "temperature": 0.1,
  "max_tokens": 128
}
```

### 7.3 远程模型期望响应格式

```json
{"一级意图": "search", "二级意图": ["专利检索"]}
```

## 八、工具执行参数

### 8.1 专利检索工具（_search_patents）

| 输入参数 | 来源 | 说明 |
|---------|------|------|
| tech_domain | state | 技术领域关键词，按空格拆分后作为检索词 |
| core_problem | state | 核心问题关键词，按空格拆分后作为检索词 |
| constraints | state | 约束条件中的字符串值作为补充检索词 |
| top_k | 参数 | 返回结果数量，默认 10 |

**检索逻辑**：对 `PATENT_DATA` 中每条专利的 `title + chunk_text + tech_field` 拼接文本进行关键词命中计分，按得分降序返回 top_k 条。

### 8.2 专利详情查询工具（_find_patent_by_id）

| 输入参数 | 来源 | 说明 |
|---------|------|------|
| patent_id | state.constraints.patent_id | 专利号，精确匹配 |

### 8.3 工具路由规则

| 意图 | 调用工具 | 返回结构 |
|------|---------|---------|
| 专利检索 / search | _search_patents(top_k=10) | `{"patents": [...], "total": N}` |
| 专利详情查询 | _find_patent_by_id | 单条专利 dict 或错误信息 |
| SWOT/对比/风险/价值 | _find_patent_by_id + 分析类型 | `{"analysis_type": "...", "patent": {...}, "tech_domain": "..."}` |
| 专利聚束组合 | _search_patents(top_k=5) | `{"bundle_name": "...", "patents": [...]}` |

## 九、完整对话流程示例

### 示例1：专利检索（多轮）

```
用户: "我想找一种耐高温的不粘锅涂层材料相关的专利"
系统: [意图=search/专利检索] → [实体抽取: 技术领域=涂层, 核心问题=耐高温不粘锅涂层]
      → [完整性评估: 缺少约束条件] → [追问]
回复: "请问您对涂层的耐温性能有具体要求吗？比如需要耐受多少度的高温？"

用户: "需要耐受400度以上，用于餐饮厨具"
系统: [意图=feedback/修改条件] → [实体抽取: 约束={temperature:400, application:餐饮厨具}]
      → [完整性评估: 通过] → [确认]
回复: "我理解您的需求是：寻找适用于餐饮厨具、耐温400℃以上的不粘涂层相关专利。请确认是否继续。"

用户: "确认"
系统: [确认通过] → [执行检索] → [生成回复]
回复: "共找到 8 条相关专利..."
```

### 示例2：专利详情查询

```
用户: "帮我查一下CN202520842474.4的详细信息"
系统: [意图=search/专利详情查询] → [实体抽取: patent_id=CN202520842474.4]
      → [直接执行] → [生成回复]
回复: "专利号 CN202520842474.4 的详细信息如下：标题:..., 发明人:..., ..."
```

### 示例3：SWOT 分析

```
用户: "对这个专利做一下SWOT分析"
系统: [意图=analysis/SWOT分析] → [实体抽取] → [直接执行] → [生成回复]
回复: "以下是该专利的 SWOT 分析报告..."
```
